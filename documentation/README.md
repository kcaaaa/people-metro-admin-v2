# 文档导航指南

## 目录结构概览

本文档提供了人民城轨2.0项目（people-metro-admin-v2）的完整文档导航指南，帮助您快速找到所需的信息。项目文档已按照逻辑层次结构组织，确保文档的清晰度、可访问性和组织性。

```
documentation/
├── 1-overview/                 # 项目概览
├── 2-requirements/             # 需求文档
├── 3-design/                   # 设计文档
│   ├── ui/                     # UI设计
│   ├── functional/             # 功能设计
│   └── data/                   # 数据设计
├── 4-implementation/           # 实现文档
├── 5-deployment/               # 部署文档
├── 6-development/              # 开发指南
├── 7-demo/                     # 演示文档
└── 8-images/                   # 图片资源
    └── 直播/                   # 直播相关图片
```

## 目录说明

### 1-overview/ - 项目概览

项目概览目录包含项目的总体介绍、变更日志和其他高级信息，帮助您快速了解项目。

- **README.md**: 项目概述、架构和主要功能
- **CHANGELOG.md**: 记录项目版本更新和功能变更历史

### 2-requirements/ - 需求文档

需求文档目录包含产品的详细需求规格说明，帮助您理解产品的目标和功能。

- **总体需求文档.md**: 描述项目的整体需求和目标
- **功能需求文档.md**: 详细描述每个功能的具体需求

### 3-design/ - 设计文档

设计文档目录包含UI设计、功能设计和数据设计的详细信息。

- **ui/**: UI设计相关文档
  - **全局UI设计文档.md**: 产品的整体UI设计规范
- **functional/**: 功能设计相关文档
  - 各功能模块的详细设计文档
- **data/**: 数据设计相关文档
  - 数据模型和数据流程设计文档

### 4-implementation/ - 实现文档

实现文档目录包含技术实现相关的文档，帮助开发人员理解如何实现需求。

- 技术实现细节、API文档和代码结构说明

### 5-deployment/ - 部署文档

部署文档目录包含项目部署相关的指南和脚本，帮助运维人员部署项目。

- **DEPLOYMENT.md**: 部署指南和说明
- **GIT部署脚本文件/**: GitHub部署相关脚本
- **部署问题记录文档.md**: 记录部署过程中遇到的问题和解决方案

### 6-development/ - 开发指南

开发指南目录包含开发环境设置、编码规范和贡献指南，帮助开发人员参与项目开发。

- **CONTRIBUTING.md**: 贡献指南
- 开发环境配置和编码规范文档

### 7-demo/ - 演示文档

演示文档目录包含项目演示相关的说明和指南，帮助用户了解如何使用项目的演示功能。

- **README.md**: 演示说明和使用指南
- **大屏管理Demo使用说明.md**: 大屏管理演示的使用说明
- **大屏可视化编辑使用指南.md**: 大屏可视化编辑功能的使用指南

### 8-images/ - 图片资源

图片资源目录包含项目文档中使用的各种图片资源。

- **直播/**: 直播功能相关的图片资源

## 导航最佳实践

- **新用户**: 建议从 `1-overview/` 目录开始，了解项目概览，然后查看 `2-requirements/` 目录中的需求文档
- **开发人员**: 建议查看 `3-design/` 目录了解设计细节，然后查看 `4-implementation/` 目录和 `6-development/` 目录
- **运维人员**: 重点关注 `5-deployment/` 目录中的部署文档
- **演示用户**: 建议直接查看 `7-demo/` 目录中的演示指南

## 文档更新和维护

项目文档会随着项目的发展不断更新。如果您发现文档中的问题或需要补充内容，请参考 `6-development/CONTRIBUTING.md` 中的贡献指南进行贡献。

每次重大更新后，我们会在 `1-overview/CHANGELOG.md` 中记录变更内容，确保您能够及时了解文档的最新变化。

## 联系我们

如果您在使用文档过程中遇到任何问题，请通过项目的GitHub仓库提交issue或联系项目维护人员。
