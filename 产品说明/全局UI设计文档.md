# 人民城轨2.0系统全局UI设计文档

## 1. 设计概述

本设计文档定义了人民城轨2.0后台管理系统的全局UI设计规范，包括整体布局、色彩方案、组件样式、交互设计等，旨在确保系统UI的一致性和用户体验的流畅性。文档涵盖AI管理、情景管理等功能模块的特定UI设计要求。

## 2. 设计原则

- **一致性**：遵循Element UI设计语言，保持与系统其他模块的视觉一致性
- **简洁性**：界面布局清晰，操作流程简单直观
- **可访问性**：确保所有用户都能方便地使用系统功能
- **响应式**：适配不同屏幕尺寸，提供良好的跨设备体验
- **视觉层次**：通过色彩、字体大小、间距等元素创建清晰的视觉层次

## 3. 布局设计

### 3.1 整体布局

AI管理功能模块采用系统统一的后台管理布局：

- **顶部导航栏**：显示系统名称、用户信息、全局操作
- **左侧菜单栏**：系统功能导航，AI管理作为一级菜单项
- **主内容区域**：显示当前功能的具体内容
- **面包屑导航**：显示当前页面的层级位置

### 5.4 知识库管理界面

#### 知识库列表页面
- **布局架构**：采用二栏布局，左侧为导航菜单，右侧为内容区域
- **顶部统计**：展示知识库总数、已使用存储空间等关键指标
- **操作区**：右上角包含"新建知识库"按钮
- **表格设计**：
  - 列定义：知识库名称、简介、文件数量、状态、创建时间、操作
  - 操作列：包含编辑、删除按钮和启用/关闭开关
- **交互设计**：
  - 点击编辑按钮：跳转到知识库详情页面
  - 启用/关闭：使用开关组件控制
  - 点击删除按钮：弹出确认对话框

#### 知识库详情页面
- **布局架构**：二栏布局，左侧导航菜单，右侧内容区域
- **顶部导航**：包含面包屑导航（AI管理 > 知识库管理 > 知识库详情）
- **基本信息区**：
  - 显示知识库名称、简介、状态等基本信息
  - 提供编辑、启用/关闭、删除等操作按钮
- **文件列表区**：
  - 表格设计：文件名称、文件类型、文件大小、创建时间、更新时间、状态、操作
  - 操作列：包含下载、删除按钮
  - 支持上传新文件
- **交互设计**：
  - 编辑按钮：弹出编辑对话框，修改知识库基本信息
  - 启用/关闭：使用开关组件控制知识库状态
  - 删除按钮：弹出确认对话框后执行删除操作
  - 上传按钮：弹出文件选择对话框，支持多文件上传
  - 下载按钮：直接下载对应文件
  - 删除文件：弹出确认对话框后删除对应文件

### 5.5 情景管理界面

#### 情景列表页面
- **布局架构**：采用二栏布局，左侧为导航菜单，右侧为内容区域
- **顶部搜索区**：
  - 搜索框：支持关键词搜索情景名称、描述等
  - 搜索按钮：带搜索图标，点击触发搜索
  - 新增按钮：用于创建新情景
- **表格设计**：
  - 列定义：情景名称、创建时间、播放形式、大屏数量、操作
  - 操作列：包含编辑、删除按钮
- **交互设计**：
  - 实时搜索：输入关键词后自动筛选结果
  - 搜索结果高亮：匹配关键词高亮显示
  - 编辑按钮：弹出编辑对话框
  - 删除按钮：弹出确认对话框

#### 情景编辑对话框
- **布局架构**：居中模态对话框
- **基本信息区**：
  - 情景名称输入框
  - 播放形式选择器（单选）：包含轮播模式和默认模式
  - 默认模式下显示默认时长输入框（带单位后缀"秒"）
- **大屏管理区**：
  - 左侧：可选择的大屏列表（复选框）
  - 右侧：已选择的大屏列表（带时长编辑）
  - 中间：添加/移除按钮（左右箭头图标）
  - 轮播模式下：每个已选大屏可单独设置播放时长
  - 时长输入框：使用输入组样式，带单位后缀
- **交互设计**：
  - 播放模式切换：切换时动态显示/隐藏相关字段
  - 时长输入验证：实时验证输入值的有效性
  - 表单验证：提交时验证必填字段和格式

## 8. 交互设计

### 8.1 按钮交互

- **点击效果**：轻微下沉效果，提供明确的点击反馈
- **悬停效果**：背景色变化，提供视觉提示
- **禁用状态**：降低透明度，禁用点击事件

### 8.2 表单交互

- **输入反馈**：实时验证，错误时显示红色边框和提示信息
- **聚焦状态**：输入框边框高亮，提供明确的焦点指示
- **下拉选择**：展开动画，选中项高亮显示

### 8.3 表格交互

- **行选中**：选中行背景色变化，提供复选框状态指示
- **排序**：点击表头可切换排序状态，显示排序图标
- **分页**：切换页码时表格数据平滑过渡

### 8.4 模态框交互

- **打开动画**：淡入效果，从中心弹出
- **关闭动画**：淡出效果
- **遮罩层**：半透明遮罩，点击可关闭模态框（可配置）

## 9. 响应式设计

### 9.1 断点设置

| 设备类型 | 屏幕宽度 | 布局调整 |
|---------|---------|----------|
| 大屏幕 | ≥1600px | 完整布局，可显示更多列 |
| 中屏幕 | 1400-1599px | 标准布局，适当减少列数 |
| 小屏幕 | 1200-1399px | 紧凑布局，隐藏非关键列 |
| 平板 | 768-1199px | 单列布局，折叠侧边栏 |
| 移动端 | <768px | 单列布局，隐藏侧边栏，使用底部导航 |

### 9.2 适配策略

- 表格在小屏幕上自动隐藏部分列，提供水平滚动
- 表单元素在小屏幕上变为单列布局
- 操作按钮在小屏幕上折叠为下拉菜单
- 模态框在小屏幕上变为全屏显示

## 10. 技术实现规范

### 10.1 组件实现

- **组件封装**：遵循单一职责原则，将UI拆分为独立可复用的组件
- **样式管理**：使用Element UI的样式变量，确保风格一致
- **图标使用**：通过`<i class="fa-solid fa-icon-name"></i>`引用FontAwesome图标
- **布局实现**：使用Element UI的栅格系统进行页面布局

### 10.2 JSX语法替代方案

为确保与现有系统的兼容性，所有组件使用React.createElement()替代JSX语法：

```javascript
// 实现示例 - 创建按钮组件
const createButton = function(text, type, onClick) {
  return React.createElement(
    'button',
    {
      className: `el-button el-button--${type || 'default'}`,
      onClick: onClick
    },
    [
      text
    ]
  );
};

// 实现示例 - 创建表格列
const createTableColumn = function(title, dataIndex, options = {}) {
  return {
    title: title,
    dataIndex: dataIndex,
    key: dataIndex,
    ...options
  };
};
```
